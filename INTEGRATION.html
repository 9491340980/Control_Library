1. Prerequisites

  • Angular 20 app (the library is built on Angular 20 — major version must match).
  • Node 18+ and npm.
  • A GitHub personal access token with read:packages + repo scopes (the package is private).

2. Step 1 — Authenticate to GitHub Packages

The package is private, so each app (and CI) must authenticate to GitHub's npm registry.

1
Create a token: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic), scopes read:packages + repo.
2
In the app's root folder, create a file named .npmrc:
@9491340980:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${NPM_TOKEN}
3
Set the token in your shell (PowerShell) before installing:
$env:NPM_TOKEN = "ghp_your_read_token"
Never paste the token inside ${ } — that means "an environment variable name". Either use ${NPM_TOKEN} + set the env var, or put the raw ghp_… value directly after = (and then add .npmrc to .gitignore).

3. Step 2 — Install

npm install @9491340980/controls

Only if you use the ch-ng-* wrapper controls (Material datepicker, ng-select, file-drop), also install their peer dependencies:

npm install @angular/material @angular/cdk @ng-select/ng-select ngx-file-drop @angular/animations

The native ch-* controls need none of these — the library stays dependency-free unless you opt into the wrappers.

4. Step 3 — Load the theme

Import the design tokens once in your global styles.scss:

/* styles.scss */
@use '@9491340980/controls/styles/theme';

If you use the ch-ng-* controls, also add their third-party CSS:

@import '@angular/material/prebuilt-themes/azure-blue.css';
@import '@ng-select/ng-select/themes/default.theme.css';
SCSS rule: all @use lines must appear before any @import or other rules, at the very top of the file.

And, only for the Material-backed ch-ng-datepicker, enable animations in app.config.ts:

import { provideAnimations } from '@angular/platform-browser/animations';

export const appConfig = {
  providers: [ provideAnimations() ],
};

5. Step 4 — Tailwind preset (optional)

If your app uses Tailwind, extend the shared preset so utilities and components share the same tokens:

// tailwind.config.js
module.exports = {
  presets: [require('@9491340980/controls/tailwind-preset')],
  content: ['./src/**/*.{html,ts}'],
};

6. Step 5 — Use the controls

All controls are standalone. Import the ones you use where you use them.

In a standalone component

import { Component } from '@angular/core';
import { FormsModule } from '@angular/forms';
import { ChButton, ChCard, ChInput } from '@9491340980/controls';

@Component({
  selector: 'app-patient',
  imports: [FormsModule, ChButton, ChCard, ChInput],
  templateUrl: './patient.html',
})
export class PatientComponent {}

In an NgModule-based app

Standalone components go in the module's imports (not declarations):

import { ChButton, ChCard, ChInput } from '@9491340980/controls';

@NgModule({
  declarations: [PatientComponent],
  imports: [FormsModule, ChButton, ChCard, ChInput],  // ← here
})
export class PatientModule {}

Template

<ch-card title="New patient">
  <ch-input [(ngModel)]="name" placeholder="Full name" />
  <ch-button variant="primary" (clicked)="save()">Save</ch-button>
</ch-card>

7. Forms, toasts & modals

Forms

Every form control implements ControlValueAccessor, so it works with template-driven and reactive forms:

<ch-input [(ngModel)]="email" type="email" />
<ch-select formControlName="country" [options]="countries" />

Toasts

Render the toaster once near the app root, then call the service anywhere:

<ch-toaster />
import { ChToastService } from '@9491340980/controls';

private toast = inject(ChToastService);
this.toast.success('Saved!');
this.toast.error('Something went wrong');

Modal

<ch-modal [(open)]="showDialog" title="Confirm">
  <p>Are you sure?</p>
  <div modal-footer>
    <ch-button variant="outline" (clicked)="showDialog=false">Cancel</ch-button>
    <ch-button variant="primary" (clicked)="confirm()">OK</ch-button>
  </div>
</ch-modal>

9. Theming & dark mode

Everything is driven by CSS variables, so theming is just overriding tokens after the theme import:

/* this app's brand */
:root { --ch-color-primary: #2563eb; }

/* dark mode */
<html data-theme="dark">

/* per-tenant */
.tenant-acme { --ch-color-primary: #16a34a; }

11. Troubleshooting

ErrorCause & fix
NG8001: 'ch-…' is not a known element The component isn't imported. Add it to the component's imports (standalone) or the module's imports (NgModule).
@use rules must be written before any other rules Move every @use to the very top of styles.scss, above all @import and CSS rules.
401 Unauthorized on install Token missing/expired, or the env var isn't set in this terminal. Run echo $env:NPM_TOKEN; set a valid read:packages+repo token. Don't wrap the token in ${ }.
404 / Cannot find package The .npmrc scope line is missing — confirm @9491340980:registry=https://npm.pkg.github.com exists in the app root.
Could not resolve "@angular/animations/browser" Install @angular/animations (needed for ch-ng-datepicker via provideAnimations()).
Controls render unstyled You didn't import the theme. Add @use '@9491340980/controls/styles/theme'; to global styles.
